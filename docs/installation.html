Installation

This guide covers setting up LinkedIn AI Publisher locally for development or preparing it for production deployment via Docker.

Prerequisites

RequirementVersionNotes
Node.js≥ 20.xLTS recommended
npm≥ 10.xBundled with Node.js
MongoDB≥ 6.xLocal or MongoDB Atlas
Redis≥ 7.xRequired for BullMQ job queues
Docker≥ 24.xOptional — for production deployment
LinkedIn Developer App—OAuth credentials required

1. Clone the Repository

# Clone
git clone https://github.com/your-username/linkedin-autopost.git
cd linkedin-autopost

# Install dependencies
npm install

2. Environment Setup

Copy the example environment file and fill in your values:

cp .env.example .env.local

See the Configuration page for a full reference of all variables.

Generate Required Secrets

# NextAuth secret (min 32 chars)
openssl rand -base64 32

# Encryption key for API key storage (must be 32 hex bytes)
openssl rand -hex 32

3. Create a LinkedIn Developer App

  1. 1
    Open LinkedIn Developer Portal

    Go to linkedin.com/developers/apps and click Create App.

  2. 2
    Configure OAuth

    Under Auth, add your redirect URI: https://yourdomain.com/api/linkedin/callback
    For local dev: http://localhost:4000/api/linkedin/callback

  3. 3
    Request Required Scopes

    Under Products, request access to: openid, profile, email, w_member_social

  4. 4
    Copy Credentials

    Copy the Client ID and Client Secret into your .env.local as LINKEDIN_CLIENT_ID and LINKEDIN_CLIENT_SECRET.

⚠️

LinkedIn app verification: The w_member_social scope (required for posting) needs LinkedIn to verify your app. During development, the app owner's account can test without verification.

4. Database Setup

Option A — Local MongoDB

# Using Docker for local MongoDB
docker run -d \
  --name mongo-dev \
  -p 27017:27017 \
  -v mongo-data:/data/db \
  mongo:latest

Option B — MongoDB Atlas (Cloud)

Create a free cluster at cloud.mongodb.com, create a database user, whitelist your IP, and copy the connection string into MONGODB_URI.

Seed the Admin User

# Creates the initial admin account from ADMIN_* env vars
npm run seed

5. Redis Setup

# Local Redis via Docker
docker run -d \
  --name redis-dev \
  -p 6379:6379 \
  redis:alpine

Set REDIS_URL=redis://localhost:6379 in your .env.local.

6. Run Locally

# Start the Next.js dev server
npm run dev

# In a separate terminal — start the BullMQ worker
npm run worker

The app will be available at http://localhost:4000.

✅

The worker process is required for: post generation, auto-publishing, campaign scheduling, and email notifications. It must run alongside the web process.

7. Set Up an AI Provider

Log in to the app, navigate to Settings → AI Provider, and configure one of the supported providers:

ProviderRequiresNotes
OpenAIAPI keyGPT-4o, GPT-4, GPT-3.5
Anthropic ClaudeAPI keyClaude 3.5 Sonnet, Claude 3
Google GeminiAPI keyGemini Pro, Gemini Flash
OllamaOllama endpoint URLSelf-hosted LLMs — Llama, Mistral, etc.
OpenRouterAPI keyAccess to 100+ models via one API

Keys are encrypted with AES-256-GCM before storage. Available models are fetched automatically from each provider.
