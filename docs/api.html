API Reference

All API routes are Next.js App Router route handlers under /src/app/api/. They require authentication via session cookie (NextAuth) unless otherwise noted.

🔐 Authentication

All API routes (except /api/auth/*) require a valid session. Sessions are established via NextAuth credential login and stored as HTTP-only cookies.

ℹ️

There is no public REST API with API keys. All endpoints are session-authenticated, designed for the web app's own frontend consumption.

POST/api/auth/signin

NextAuth credential sign-in. Body: { email, password }

POST/api/auth/signout

Sign out and clear session cookie.

💡 Content API

Discover content ideas

POST/api/content/discover

Takes pasted text and returns 6 AI-generated content ideas.

Request Body 
{
  "sourceContent": "Paste any text here..."
}
Response 
{
  "success": true,
  "ideas": [
    {
      "title": "Why AI is reshaping content marketing",
      "angle": "Contrarian take on AI content tools",
      "hook": "Everyone says AI will replace writers. They're half right.",
      "tags": ["AI", "ContentMarketing", "Writing"]
    }
    // ... 5 more
  ]
}

I Read — article-to-posts

POST/api/content/iread

Takes article text and returns 3 complete LinkedIn posts from 3 angles.

Request Body 
{
  "content": "Full article text...",
  "tone": "Conversational"  // optional
}
Response 
{
  "success": true,
  "posts": [
    {
      "angle": "key_insight",
      "angleLabel": "Key Insight",
      "summary": "The most surprising finding from this research",
      "content": "I just read a fascinating study on...",
      "hashtags": ["Research", "Learning"]
    },
    // practical_application, nuanced_take
  ]
}

      },

      "schedule": {
        "frequency": "weekly",
        "daysOfWeek": [1, 3],
        "time": "09:00",
        "timezone": "Asia/Dhaka",

      "approvalMode": "email",
      "isActive": true,
      "postsGenerated": 12,
      "lastRunAt": "2026-06-09T09:00:00.000Z"
    }

Create campaign

POST/api/campaigns
Request Body 
{
  "name": "Weekly AI Insights",
  "niche": {
    "industry": "Technology",
    "topics": ["Artificial Intelligence", "LLMs"],
    "keywords": ["AI", "automation"]
  },
  "tone": "Educational",
  "examplePosts": ["Example post 1...", "Example post 2..."],
  "schedule": {
    "frequency": "weekly",
    "daysOfWeek": [1, 3],
    "time": "09:00",
    "timezone": "Asia/Dhaka"
  },
  "approvalMode": "email",
  "isActive": true
}

Update campaign

PATCH/api/campaigns/:id

Update any editable field. Niche fields are ignored — they cannot be changed after creation.

⚠️

Changing isActive from false to true will immediately add a BullMQ repeating job. Changing from true to false will remove it.

Delete campaign

DELETE/api/campaigns/:id

Deletes the campaign and cancels the repeating BullMQ cron job.

📄 Posts API

Generate posts (via AI)

POST/api/posts

Queues an AI content generation job. Posts appear in Drafts when complete.

Request Body 
{
  "topic": "Why remote work changed team dynamics",
  "keywords": ["remote", "leadership"],
  "tone": "Conversational",
  "count": 1
}

Save draft (no AI generation)

POST/api/posts/draft

Saves pre-written content directly as a draft — bypasses the generation queue. Used by the I Read feature.

Request Body 
{
  "content": "Full LinkedIn post text...",
  "hashtags": ["tag1", "tag2"]
}

Get / Update / Delete a post

GET/api/posts/:id
PATCH/api/posts/:id
DELETE/api/posts/:id

PATCH supports updating content, hashtags, status, rejectionReason.

Publish a post immediately

POST/api/posts/:id/publish
Request Body 
{
  "action": "publish_now"
  // or
  "action": "schedule",
  "scheduledAt": "2026-06-15T09:00:00.000Z"
}

Sync published post status

POST/api/posts/sync

Checks LinkedIn API to verify if a post still exists. Updates status to deleted if LinkedIn has removed it.

🤖 AI API

GET/api/ai/settings

Get the current user's active AI provider settings (API key is masked).

POST/api/ai/settings

Save AI provider configuration. API key is AES-256-GCM encrypted before storage.

GET/api/ai/models

Fetch available models from the configured AI provider.

POST/api/ai/test

Test the AI provider connection by sending a short test prompt.

POST/api/ai/chat

Direct chat with the configured AI provider. Used internally by generation features.

🔗 LinkedIn API

GET/api/linkedin/connect

Initiates the LinkedIn OAuth flow. Redirects to LinkedIn's authorization page.

GET/api/linkedin/callback

OAuth callback. Exchanges the auth code for an access token, stores encrypted token. Redirects to /settings?tab=linkedin.

🔔 Notifications API

GET/api/notifications

List all notifications for the current user. Supports ?unread=true filter.

PATCH/api/notifications

Mark notifications as read. Body: { "ids": ["id1", "id2"] } or { "all": true }.

🛡️ Admin API

All admin endpoints require role: "admin" on the session user.

GET/api/admin/stats

Platform-wide statistics: user count, posts generated, active LinkedIn accounts, etc.

GET/api/admin/users
PATCH/api/admin/users/:id

Manage user accounts — list all users, update status (active/suspended).

GET/api/admin/logs

Activity audit log for all user actions across the platform.

🛠️ Developer API

GET/api/developer/syslogs

Stream of server-side console logs stored in Redis. Used by the Developer → System Logs page.

GET/api/developer/queues

BullMQ queue statistics — active, waiting, completed, and failed job counts per queue.

GET/api/dashboard

Dashboard statistics: recent posts, counts by status, recent activity timeline.
