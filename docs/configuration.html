Docs/Configuration

Configuration

All configuration is done via environment variables. Copy .env.example to .env.production (for Docker) or .env.local (for local development) and fill in your values.

⚠️

Never commit .env.production or .env.local to version control. Both are in .gitignore by default.

Application

VariableDefaultRequiredDescription
NEXT_PUBLIC_APP_URLhttp://localhost:4000YesPublic URL of the app — used in emails, OAuth callbacks, and LinkedIn redirect URIs
NEXT_PUBLIC_APP_NAMELinkedIn AI PublisherNoApp name shown in the UI and emails
NODE_ENVdevelopmentNoSet to production in production environments

Authentication (NextAuth)

VariableRequiredDescription
NEXTAUTH_URLYesMust match NEXT_PUBLIC_APP_URL. Used by NextAuth for callback URLs.
NEXTAUTH_SECRETYesMin 32-character random string. Generate: openssl rand -base64 32

Admin Bootstrap

These are used by npm run seed to create the initial admin account. Only used once on first setup.

VariableDescription
ADMIN_EMAILAdmin login email
ADMIN_PASSWORDAdmin password (min 8 chars, must include uppercase + number + symbol)
ADMIN_NAMEAdmin display name

MongoDB

VariableRequiredExample
MONGODB_URIYesmongodb://localhost:27017/linkedin-ai-publisher
ℹ️

In Docker Compose, the web container overrides this to mongodb://mongodb:27017/linkedin-autopost using the internal Docker network hostname.

Redis

VariableRequiredExample
REDIS_URLYesredis://localhost:6379

LinkedIn OAuth

VariableRequiredDescription
LINKEDIN_CLIENT_IDYesFrom LinkedIn Developer App → Auth tab
LINKEDIN_CLIENT_SECRETYesFrom LinkedIn Developer App → Auth tab
LINKEDIN_REDIRECT_URIYesMust match exactly: {APP_URL}/api/linkedin/callback

Required OAuth Scopes

Your LinkedIn app must have access to these scopes (granted via LinkedIn's Products section):

openid
profile
email
w_member_social

Encryption

VariableRequiredDescription
ENCRYPTION_KEYYes32-byte hex string (64 hex chars). Used for AES-256-GCM encryption of AI provider API keys and LinkedIn tokens. Generate: openssl rand -hex 32
🔴

Critical: Changing the encryption key after users have saved API keys will make those keys unreadable. Back this up securely and never rotate it without a migration.

Email / SMTP

Used for campaign approval emails and notification emails. Set SMTP_ENABLED=false to disable all outbound email.

VariableDefaultDescription
SMTP_ENABLEDfalseSet to true to enable email sending
SMTP_HOSTsmtp.gmail.comSMTP server hostname
SMTP_PORT587SMTP port (587 for STARTTLS, 465 for SSL)
SMTP_SECUREfalseSet true for port 465 (SSL)
SMTP_USER—SMTP authentication username
SMTP_PASS—SMTP password or app password
SMTP_FROM—From address, e.g. "LinkedIn AI Publisher <noreply@domain.com>"

Gmail Setup

SMTP_ENABLED=true
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your@gmail.com
SMTP_PASS=xxxx-xxxx-xxxx-xxxx   # Gmail App Password (not your login password)
SMTP_FROM="LinkedIn AI Publisher <your@gmail.com>"

AI Providers (Global Fallbacks)

Users configure their own API keys per-account in Settings. These server-level variables act as optional global fallbacks only.

VariableProvider
OPENAI_API_KEYOpenAI (GPT-4, GPT-3.5)
ANTHROPIC_API_KEYAnthropic Claude
GEMINI_API_KEYGoogle Gemini
OLLAMA_BASE_URLOllama (local LLMs) — e.g. http://localhost:11434
OPENROUTER_API_KEYOpenRouter (100+ models)

Background Workers

VariableDefaultDescription
WORKER_CONCURRENCY5Max concurrent BullMQ jobs
AUTO_PUBLISH_DELAY_HOURS24Hours to wait before auto-publishing a draft (if auto-publish is enabled)
TOKEN_REFRESH_INTERVAL_HOURS6How often to refresh LinkedIn OAuth tokens

Rate Limiting

VariableDefaultDescription
RATE_LIMIT_MAX100Max requests per window per IP
RATE_LIMIT_WINDOW_MS900000Window in milliseconds (900000 = 15 minutes)

Complete Example

📄 .env.production 
# App
NEXT_PUBLIC_APP_URL=https://linkedinpost.example.com
NEXT_PUBLIC_APP_NAME="LinkedIn AI Publisher"
NODE_ENV=production

# Auth
NEXTAUTH_URL=https://linkedinpost.example.com
NEXTAUTH_SECRET=your-super-secret-min-32-chars

# Admin (used only for seeding)
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=Admin@123456
ADMIN_NAME="System Administrator"

# LinkedIn OAuth
LINKEDIN_CLIENT_ID=86xxxxxxxxxx
LINKEDIN_CLIENT_SECRET=xxxxxxxxxxxxxxxx
LINKEDIN_REDIRECT_URI=https://linkedinpost.example.com/api/linkedin/callback

# Encryption (openssl rand -hex 32)
ENCRYPTION_KEY=a3f9c2b8e1d7...

# Infrastructure (overridden by docker-compose)
MONGODB_URI=mongodb://localhost:27017/linkedin-autopost
REDIS_URL=redis://localhost:6379

# Email
SMTP_ENABLED=true
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=you@gmail.com
SMTP_PASS=xxxx-xxxx-xxxx-xxxx
SMTP_FROM="LinkedIn AI Publisher <noreply@example.com>"

# Workers
WORKER_CONCURRENCY=5
AUTO_PUBLISH_DELAY_HOURS=24
TOKEN_REFRESH_INTERVAL_HOURS=6
